Viridis: Predictive Reporting and User Interface

The Viridis project was a team-based endeavor where we worked with a client to understand their requirements and create a scalable solution. I was part of a 6-7 member team and actively contributed to the implementation of key features. The goal was to develop a platform that would allow staff to generate predictive reports and insights, ultimately enhancing customer presentations.

"A scalable solution to enhance predictive reporting and data-driven insights for customer presentations."

Project Methodology

In the first semester, our team worked closely with the client to gather requirements, which involved both online and offline meetings. We created mockups and refined them based on feedback from the client to better understand their needs. We used these insights to create a robust platform focused on generating predictive reports and providing insights that were easy to interpret.

My Role & Contributions

I led the implementation of several key features in the project, including:

  • Secure User Authentication: Ensured that only authorized users could access sensitive data.
  • Robust Database Management: Created and managed the database to support the platform's scalability.
  • Dynamic Administrator Interface: Developed an interface that improved user experience by providing quick access to data and graphical reports.

Future Improvements

  • Enhanced Data Visualization: Expanding the graphical reports to include more interactive features for users.
  • Advanced Predictive Analytics: Implementing machine learning algorithms for more accurate predictions.

Hackcessible

This project, part of the Hackcessible event, involved creating assistive technology to help an individual enjoy playing drums. The main goal was to design and build a system using Arduino to enhance accessibility and provide a solution for physical challenges. The project was a collaborative effort, and I had the opportunity to apply my skills in coding and problem-solving.

Hackcessible
"A great step toward making music more accessible for everyone."

Methodology

Our approach focused on using Arduino to create a device that would support the individual in interacting with musical instruments. As part of the team, I contributed by suggesting coding solutions and exploring the full potential of Arduino to achieve the desired functionality.

Future Work / Improvements

The project showcased the potential of combining accessibility with technology. Moving forward, there are several areas for improvement:

  • Refining Hardware Integration: Enhancing the device’s hardware setup for improved durability and responsiveness.
  • Optimizing Code Efficiency: Further refining the Arduino code to optimize performance and reduce potential latency.
  • Expanding Accessibility Features: Exploring additional accessibility features that could benefit users with different needs, including more customizable inputs.
  • Testing and Feedback: Collecting feedback from users to iterate and improve the device for better usability and comfort.

These improvements would help enhance the device’s impact and make it more effective for users with accessibility challenges.

Project Timeline

The project progressed through various stages, including brainstorming, hardware setup, coding, testing, and refinement. Despite facing challenges, the team successfully built a functional prototype that helped us win first place in the event.

Garden Planning Add-on

The Garden Planning Add-on for Blender enables users to simulate the growth of various plants through computational models. Using Python scripting in Blender, this add-on creates dynamic plant structures, simulating growth patterns for plants like sunflowers and baby's breath. The add-on leverages L-systems and randomization to generate plant branching, offering a glimpse into nature's beauty through code.

ZuhairaTech
"Blending computational algorithms with nature's intricate design."

Methodology

In this project, L-systems were adapted to generate plant growth models in Blender using Python scripting. Growth patterns such as branching angles, plant height, and radius were controlled based on predefined rules for each plant type such as below:


            plants_rules = {
                "SUNFLOWER": {
                        "angle_of_branching": 0,  # degrees
                        "radius": 0.1,
                        "height": 0.5,
                    },
                "BABYSBREATH": {
                        "angle_of_branching": 1,  # degrees
                        "radius": 0.01,
                        "height": 0.2,
                    },
                # Add more plant types as needed
            }

Refer: garden_planning.py

Project Summary

This project explores a simplified approach to rule-based branching inspired by L-systems, adapted for easier use with Blender and Python. It focuses on developing a tool for garden planning that allows users to design and visualise layouts efficiently while maintaining a user-friendly experience. Blender was used for 3D modelling and Python for scripting. Although the tool is functional, it remains a work in progress, with future updates aimed at enhancing its capabilities and performance in more complex settings. The modular nature of the system makes it adaptable for other uses beyond garden planning. Planned improvements include refining plant models—particularly for sunflowers and baby's breath—by using more realistic growth rules, incorporating environmental factors, expanding the use of L-systems to simulate various natural forms, improving randomness to better reflect natural growth, and enhancing 3D visualisation and overall performance for larger simulations.
